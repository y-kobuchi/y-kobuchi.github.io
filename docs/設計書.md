# Next.js ポートフォリオサイト 設計書

## 1. 概要

本設計書は、Next.js（App Router）を使用して構築するポートフォリオサイトの設計をまとめたものである。サイトは日本語を中心とし、落ち着いた男性的なデザインを採用する。利用技術は Next.js、TypeScript、Tailwind CSS、shadcn/ui を基盤とする。

---

## 2. 目的

* フリーランスエンジニアとしての実績・スキルを分かりやすく提示する。
* クライアント・エージェントへの信頼性向上。
* 作業経歴や成果物を整理し、案件獲得率の向上を図る。

---

## 3. ターゲットユーザー

* エンジニア系エージェント
* 企業の採用担当者
* 技術者コミュニティに所属する閲覧者（GitHub 経由など）

---

## 4. 使用技術

* **Next.js 16（App Router）**
* **TypeScript**
* **Tailwind CSS**
* **Vercel（ホスティング）**

---

## 5. サイト構成

```
src/
 └ app/
     ├ layout.tsx
     ├ page.tsx              # トップページ
     ├ about/
     │   └ page.tsx
     ├ skills/
     │   └ page.tsx
     ├ works/
     │   └ page.tsx
     ├ contact/
     │   └ page.tsx
     └ components/           # 共通コンポーネント
         ├ Header.tsx
         ├ Footer.tsx
         ├ SectionTitle.tsx
         ├ WorkCard.tsx
         └ SkillCard.tsx
```

---

## 6. デザイン方針

### 色設計

* **基調色（背景）：** `#0f172a`（濃紺）
* **アクセント：** `#22c55e`（緑）
* **テキスト：** `#e2e8f0`（淡いグレー）

### UI/UX

* 男性的でシャープな印象
* グラデーションはヘッダー・フッターのみに使用
* 角丸は控えめ（`rounded-lg` 程度）
* アニメーションは軽めにし、視認性を優先

---

## 7. 各ページの仕様

### 7.1 トップページ

**目的：** 第一印象で技術力と信頼感を伝える。
**要素：**

* メインビジュアル（名前・肩書き）
* キャッチコピー
* スキル一覧のダイジェスト（AWS、Python、TypeScript）
* 実績ダイジェスト（主要プロジェクト3件）
* Works ページ・Contact ページへの導線

### 7.2 About ページ

**目的：** 経歴・人物像を詳しく説明。
**内容：**

* 職務要約（AWS 4年、フリーランス）
* 経歴（フリーランス、シーズ・ラボ、新人研修）
* エンジニアとしての強み（AWS サービス経験、新技術への適応力、コード品質）
* 資格（AWS Certified Cloud Practitioner、基本情報技術者）

### 7.3 Skills ページ

**目的：** 使用技術を可視化。
**内容：**

* プログラミング言語（Python、TypeScript、JavaScript、ShellScript）
* AWS サービス（Lambda、API Gateway、CloudFormation、DynamoDB、RDS、S3 等）
* データベース（PostgreSQL、MySQL、DynamoDB、SQLite）
* フロントエンド（Angular、HTML/CSS、React）
* DevOps/インフラ（Terraform、Docker、CI/CD、Linux）
* スキルレベルの定義説明

### 7.4 Works ページ

**目的：** 実績を詳細に掲載。
**内容：**

* プロジェクト5件
  * 幼稚園バス見守りシステム アカウント移行
  * BtoB バスロケツール開発
  * BtoC ドラレコ開発
  * 研究開発 バス乗客人物判定
  * 研究開発 AI 利用システム
* WorkCard でプロジェクト一覧表示
* 担当範囲 / 技術スタック / 工夫した点
* GitHub リンク

### 7.5 Contact ページ

**目的：** 仕事依頼への導線。
**内容：**

* 連絡先情報（Email、GitHub、LinkedIn）
* お問い合わせフォーム（名前、メールアドレス、件名、メッセージ）
* 返信時間の目安記載

---

## 8. コンポーネント設計

### 8.1 Header

* グラデーション背景（濃紺から濃灰へ）
* ナビゲーションリンク（ホーム、About、Skills、Works、Contact）
* sticky 位置付け
* Yuma Kobuchi ロゴ表示

### 8.2 Footer

* グラデーション背景（Header と共通）
* コピーライト記載

### 8.3 SectionTitle

* セクションタイトル
* サブタイトル（オプション）
* アクセント線

### 8.4 WorkCard

* プロジェクト名
* 説明
* 工夫した点
* 技術スタック
* GitHub リンク

### 8.5 SkillCard

* 技術名
* 熟練度（5段階レベルバー）

---

## 9. ページ遷移・ルーティング図

* `/` トップページ

  * `/skills` Skills ページ
  * `/works` Works ページ
